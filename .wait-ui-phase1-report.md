# Wait-UI Phase 1 Report

## Wire shape emitted

```json
{
  "wait": {
    "expr_cel": "signal_received",
    "phase": "waiting",
    "terms": [
      {
        "name": "sig1",
        "kind": "signal",
        "label": "My Signal",
        "signal_key": "my-signal"
      }
    ],
    "inputs": { "deps": [], "signals": [], "timers": [] },
    "started_at": "2026-06-23T10:00:00Z"
  },
  "wait_reason": "wait"
}
```

Fields omitted when absent: `wait` (omitempty on task), `expr_cel`/`signal_key`/`timer_name` on terms (omitempty), `resolved_at`/`started_at`/`summary` (omitempty). `inputs` always present with non-nil empty arrays. `terms` always a non-nil slice. `evidence`/`result` omitted (Phase 2).

## wait_reason logic

- Non-pending → `"none"` always.
- Pending: `hasWait` = wait present AND phase != "resolved"; `hasIncompleteDeps` = any declared dep whose sibling state is not completed/cancelled+ignoreCancelled/discarded+ignoreDiscarded/missing+ignoreDeleted.
  - Both → `"dependencies_and_wait"`
  - Wait only → `"wait"`
  - Deps only → `"dependencies"`
  - Neither → `"none"`

## wait_reason scenarios covered

| Scenario | Expected |
|---|---|
| Pending, wait only (signal, no deps) | `"wait"` |
| Pending, wait + incomplete dep (running sibling) | `"dependencies_and_wait"` |
| Completed task with resolved wait | `"none"` |
| Pending, dep is completed (satisfied) | `"none"` |
| Non-pending (running, cancelled) | `"none"` |
| Pending, incomplete dep (no wait) | `"dependencies"` |

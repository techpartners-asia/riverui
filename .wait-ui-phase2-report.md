# Wait-UI Phase 2 Report

## Routes added

- `GET /api/pro/workflows/{id}/task-signals` — lists workflow signals from the driver, with `key` filter, `desc` ordering (default true = newest-first), `limit`+1 pagination producing `has_more`/`next_cursor_id`. Query params `task_name`, `scope`, `term_name`, `workflow_attempt`, `cursor_id` are accepted without error; `task_name` is required.
- `GET /api/pro/workflows/{id}/task-wait-diagnostics` — evaluates the wait expression via `riverworkflow.WaitDiagnosticsForExec`. Requires `task_name`.

## Wire shapes

### task-signals
```json
{"signals":[{"attempt":0,"created_at":"...","id":42,"key":"order.ready","payload":{...},"source":null}],"has_more":false,"next_cursor_id":null,"scope":"history"}
```
`id` is serialized as `int64String` (number) to match `WorkflowTaskSignalFromAPI.id: number | string`. `attempt` is always 0 (WorkflowSignal has no per-signal attempt counter — see below). `source` is null when absent.

### task-wait-diagnostics
```json
{"expr_result":false,"inputs":{"deps":[],"signals":[],"timers":[]},"inspected_at":"2026-06-24T...","phase":"waiting","signal_scan_count":0,"signal_scan_limit":10000,"terms":[{"last_matched_id":null,"matched_count":0,"name":"sig1","required_count":0,"satisfied":false}],"truncated":false,"workflow_attempt":0}
```

## Diagnostics impedance gap (WaitDiagnostics vs frontend)

The library `WaitTermDiagnostic` has `Name`, `Kind`, `Label`, `Result`, `Detail` — no per-term `matched_count`, `required_count`, `last_matched_id`. Mapped `Result` → `satisfied`; zero-defaulted the count/ID fields with a Go comment `// NOTE: richer per-term match counts require WaitDiagnostics to expose them (follow-up)`.

`inputs` (deps/signals/timers) are empty arrays — `WaitDiagnostics` does not expose the per-input state it loaded internally. Noted with `// NOTE: richer diagnostics inputs require WaitDiagnostics to expose them (follow-up)`.

`signal_scan_count` = 0 (not exposed by library). `workflow_attempt` = 0 (not in scope). `eval_error` omitted. `inspected_at` = `time.Now().UTC()` at call time.

Phase mapping: `WaitPhasePending` → `"waiting"`, `WaitPhaseResolved` → `"resolved"`, `WaitPhaseNoWait` → `"not_started"`, else `"unknown"`.

`attempt` on signals: `WorkflowSignal` has no attempt counter; field is 0. Frontend accepts `number`, so 0 is a valid no-op value.
